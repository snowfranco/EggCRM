# Pipeline Management

Pipeline management requires the **Professional** plan or higher. It is not available on Starter.

## Pipelines and stages
A pipeline is an ordered set of **stages** that a deal moves through (for example: Lead →
Qualified → Proposal → Negotiation → Won/Lost). Create and reorder pipelines under
**Settings > Pipelines**. You can run multiple pipelines (for example, one for New Business and
one for Renewals); each deal belongs to exactly one pipeline at a time.

## Working with deals
Create a deal from **Deals > New Deal** or from a contact's timeline. A deal stores an amount,
owner, associated contact and company, expected close date, and current stage. Drag deals between
stages on the board view, or update the stage field directly. Stage changes are written to the
deal's history and to the associated contact's timeline.

## Deal probability and forecasting
Each stage can carry a default **win probability** (0–100%). Forecast value is the deal amount
multiplied by stage probability; the **Forecast** report sums weighted value by close date.
Advanced forecasting visualizations are part of **advanced analytics**, which is Enterprise-only —
basic forecasting is available on Professional.

## Automating pipeline movement
With **workflow automation** (Professional+) you can trigger actions when a deal changes stage —
for example, create a task, send a templated email, or notify a Slack channel. See **Workflow
Automation** for building these rules.
